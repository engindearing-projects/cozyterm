The only coding agent that learns from you

Other AI coding tools ship a frozen model. You use it, they get your data, and the model never changes. CozyTerm is different.

The Forge pipeline captures every coding session as training data, fine-tunes your local models, and deploys them back — automatically. The more you code, the better your agent gets. Your data stays on your machine. Your models improve for you.

Multi-model architecture

Each task is routed to a specialist model. The orchestrator handles tool calling while domain experts handle the rest.

Orchestrator
qwen2.5:7b
Tool calling and agent loop. Drives the read-edit-verify cycle.
Coder
engie-coder
Code generation and editing. Fine-tuned on real coding sessions via Forge.
Reasoner
glm-4.7-flash
Deep analysis, planning, debugging. The thinking model.
Chat
llama3.2
Conversation and quick answers. Fast context lookup.

All models run locally via Ollama. Swap any model in .cozyterm.json. Falls back to Anthropic API when Ollama is unavailable.

Features

Everything you need to code faster from the terminal.

⚙ 8 built-in tools

Read, write, edit, bash, glob, grep, undo, and diagnostics. The agent reads your code, makes changes, runs commands, and self-corrects when LSP reports errors.

⚡ LSP integration

Auto-detects TypeScript, Python, Go, and Rust. After each edit, the agent checks for diagnostics and fixes its own mistakes before you see them.

🔒 Permission system

Interactive prompts for risky tools. Allow once, always, or deny forever. Persistent rules saved locally. Or --yes to skip.

🔌 MCP support

Connect external tools via Model Context Protocol. Define servers in .cozyterm.json and the agent discovers their tools at startup.

🎨 Themes

Three built-in themes: cozyterm (warm amber), dracula, and tokyonight. Switch with --theme or set in config.

📋 Custom commands

Create reusable prompts as markdown files in .cozyterm/commands/. Run them with /review, /test, or any name you choose.

Forge™: self-improving models

Every coding session generates training data. Forge collects, trains, and deploys better model versions automatically. This is the CozyTerm moat — the more you use it, the smarter it gets.

Install

One command. The installer handles Bun, clones the repo, and sets up cozy in your PATH.

2

Pull models

Install Ollama, then ollama pull qwen2.5:7b-instruct and any other models you want.

Code

cozy for the interactive TUI. cozy "fix the bug" for one-shot. cozy --plan for read-only analysis.

CozyTerm is built from scratch. No OpenClaw, no LangChain, no AutoGen, no third-party agent frameworks. The entire agent loop, tool system, provider abstraction, TUI, and training pipeline are original TypeScript written by engindearing. We took architectural inspiration from open-source projects (as any good project does), but every line of code is ours.
